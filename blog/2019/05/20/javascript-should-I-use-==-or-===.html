What is JavaScript?

  • High level dynamic, untyped and interpreted programming language, alongside HTML and CSS is one of the 3 core technologies of world wide web content
  • Conforms to the ECMAScript specification
    • Consider a modern web page with great content, fancy design, and layout, and advanced interactivity. See images: the combined output of 3 distinct layers of code, put together by the browser.
    FIRST LAYER
    HTML document, this is where the content lives, marked up in a language that makes it easy for the browser to parse. When you visit the webpage, you are accessing this doc. and without this HTML doc, there's no webpage.



    SECOND LAYER
    A collection of CSS rules. This is the code that instruct the browser how to display the HTML markup. HTML and CSS work together to ensure a perfect separation of concerns. HTML handles the content, CSS is presentational enhancement.



    THIRD LAYER
    JavaScript, interactive layer. It runs in the browser and interacts with HTML markup and CSS rules to change what you see and what you can do.



  • Multi-paradigm meaning you can write your code in many different ways, many different paradigms. For instance, you can write onject-oriented code or functional code, compares to some languages where you have to write code in a certain way.
  • Runs on the client/browser as well as on the server (Node.js)
    Your web browser displays HTML markup with CSS styling, but if you want interactive aspects on your page, like form validation, alerts, etc., then you can use JavaScript for those stuff. JavaScript is the language of front-end, meaning the browser, but you can run JavaScript on the server for more powerful things like interacting with the databases, using a JavaScript runtime called Node.js.

There are 3 ways in JavaScript to set variable
// var, let, const

  • var: has been used since the beginning of JavaScript. However, it's recommended not to use it anymore because var is globally scoped, as now we have let and const that are added to ES6 (ES2015). For example, if we have a conditional like an if statement and we set a variable inside of that block, then outside of that block of the if statement, there's another variable with that same name, that can be a conflict and it can cause problem(s).
  • let: it lets you re-assign values
    Example:
    let age = 26;
    age = 30;
  • const: const is short for constant, meaning it can't be changed, can't be directly re-assigned. If you declare a constant like const firstName;, this will give you a SyntaxError: Missing initializer in const declaration, because you have to add a value if you use const. So, const firstName = 'Kim'; is correct. Then you can just print it out, say to console log using template string/literal in ES6 with backticks, money sign, and curly braces. console.log(`My name is ${firstName}.`);
    For that same example above using const:
    let age = 26;
    age = 30; If you try to do this with const, you will get a TypeError: Assignment to constant variable.
  • So when should I use let and when should I use const ?
    It's different for everyone, some people will just use let, but the best practice I found while learning to code is always use CONST, unless you know that you are going to re-assign the value. I think this will make your code more robust, more secure, less prone to errors.

== or === ?

  • If you use double equal signs, it won't match the datatypes, it will just match the value. For example,
    const x = '10';
    if(x == 10) {
    console.log('x is 10');
    } else {
    console.log('x is NOT 10');
    }
    So, you will get x = 10 as answers for both 10 = 10 and '10' = 10 if you use ==, but if you use triple equal signs ===, you will get x is NOT 10, because it also matches the datatypes (as one is a string, and the other one is a number).
  • Different developers will treat this differently, they will decide to pick and to choose which one to use, as a newbie like myself, I like to always use the triple equals === because for the most part, always want to match the types.
