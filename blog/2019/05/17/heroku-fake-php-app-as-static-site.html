A "PHP app" as a static site that is hosted on Heroku

What is Heroku? Why do this?

  • Heroku hosts apps on the internet, not static website. However, if you want it to run your static porfolio, personal blog, responsive landing page, etc., you can trick Heroku to think your page is PHP app.

Assumption

  • You want to deploy a straight-up HTML, CSS, JavaScript, jQuery, and maybe some images
  • You are in the root directory of your site (i.e. the directory that contains all subdirectories and files for the site.
  • Unless you have all of your subdirectories for CSS, JS, etc., and files like HTML in /home or /web as the root directory, then see additional instruction when create Procfile
  • The root directory contains a main HTML page, e.g. index.html
  • You have an account with Heroku
  • You have Heroku CLI installed on your machine.
  • A Heroku app (in this case, "fake" PHP app (your HTML/CSS static site) and remote are set-up and ready to go.

Instruction Steps

  1. Run cmd, type: heroku buildpacks:set heroku/php --Read more: Heroku buildpacks
  2. Add a file called composer.json to the root directory by running cmd or directly from your project folder that you have it opened in IDE --type: touch composer.json or --on command window, type: type nul > composer.json (this will create a zero bytes in composer.json file) or another way, you type: echo.> composer.json (echo. will create a file with 1 empty line in it)



  3. Open composer.json and add the following line: {}, then save file



  4. Add a file called index.php to the root directory by running cmd: touch index.php or nul > index.php
  5. Rename/Refactor your project's homepage (e.g. index.html) to home.html
  6. Open index.php and add the following line: <?php include_once("home.html"); ?> , then save file



  7. Add a file called Procfile without a file extension, e.g. Procfile.text is not valid to your app's root directory. It does not function if placed anywhere else.
  8. Open Procfile and add the following line: web: vendor/bin/heroku-php-apache2. Notice: if your root directory is /web or /home, then add the following line: web: vendor/bin/heroku-php-apache2 /web or web: vendor/bin/heroku-php-apache2 /home



  9. Save everything and run Heroku CLI
    1. heroku login sign in to your Heroku account



    2. cd /your project path
    3. git init initialize an empty Heroku git in your application folder
    4. heroku git:remote -a your-app-name remote to the app you created on your Heroku account
    5. git add . this will add all files in your folder to Heroku
    6. git commit -am "initial commit" commit code (permanently save all the changes which you have made in the current transaction) with a message, if you don't provide a message, your commit will be aborted (to disrupt the action by killing the current ongoing transaction).
    7. git push heroku master this will push all your code to master branch
    8. heroku open this command will open your heroku site on your windows default browser

Dependencies

For example:

  • If your app runs jQuery, then you can go to CNDJS and search for jquery, CND will give you a list of results that it found for the library that your app depends on, hover over the one you want to copy script tag, then place it inside your HTML file. Benefits? jQuery library will be pulled from the internet directly versus local file.
