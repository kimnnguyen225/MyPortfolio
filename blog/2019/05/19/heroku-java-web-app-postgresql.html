Host Java Web Application with PostgreSQL Database to Heroku

What is Heroku? What is free dyno hour?

  • Heroku hosts apps on the internet using Free Dyno Hours launched. Heroku accounts have an account-based pool of free dyno hours for use on free apps. This replaces the 18 hours a day limit on free apps, allowing a free app to run 24/7 if needed. New accounts receive 550 free dyno hours and you can verify your identity with a credit card for an additional 450 hours.

Follow this guy "Java master" 's instructions

  1. Create a new app on Heroku



  2. Then click on Resources



  3. Then click on Find more add-ons



  4. Heroku will open a new window page and scroll down to look for Heroku Postgres. You can use JawsDB MySQL (as I prefer to use it) but it asks you to enter your credit card information to verify your account/user, Heroku Postgres doesn't require that.



  5. Click on Install Heroku Postgres



  6. Under App to provision to, type your app's name that you just created



  7. Click on Provision add-on button



  8. Click on Heroku Postgres



  9. Step 1: Click on Settings
    Step 2: Click on View Credentials...



  10. This is your app's database credentials, use these info to enter in SQL Shell (psql)



  11. Because I am new to PostgreSQL, so...
    In SQL Shell (psql), when creating a new table... I run into syntax errors (postgre's keywords)
    To use the "word/name" you want for your table, row/data, column names, use double quotes. In my example below, I created a table named user, in SQL Shell, I have to type CREATE TABLE "user"(username text, password text, firstName text, lastName text);, and make sure you end your sql with ;
    So here is the Postgre keywords list > SQL Key Words





Configure pom.xml & create a new Procfile for Heroku

  • Add these lines to your pom.xml
    <dependency>
    <groupId>org.postgresql</groupId>
    <artifactId>postgresql</artifactId>
    <version>9.4.1208</version>
    </dependency>
    And the plugin. Code here



  • Add a file called Procfile without a file extension, e.g. Procfile.text is not valid to your app's root directory. It does not function if placed anywhere else. Open Procfile and add the following line:
    web: java $JAVA_OPTS -jar target/dependency/webapp-runner.jar --port $PORT target/*.war
