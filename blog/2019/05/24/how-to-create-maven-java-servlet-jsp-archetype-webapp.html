How to create Java JSP Servlet with Maven Archetype Webapp

Disclaimer


The information in this guide is true and complete to the best of the author's knowledge. All recommendations are made without guarantee on the part of the author. By using this guide you understand that the author is not responsible to troubleshoot your application.

Audience


This guide is intended to provide a how-to guide for beginner Software Developers.

You will need:


  • IDE, I'm using IntelliJ IDEA Ultimate (paid) 2019 version
  • Basic knowledge or familiar with war and war_exploded of IntelliJ
  • Java JDK, I'm using 12.0.1 version
  • Basic knowledge of Tomcat server, I'm using 9.0.19 version
  • Basic knowledge of how to create JSP/Servlet as you would with Java EE
  • Helpful if have basic knowledge with MVC (Model View Controller)


Let's begin...

    1. Create new Java project
    2. Choose Maven
    3. Click on check-box Create from archetype
    4. Look for org.apache.maven.archetypes:maven-archetype-webapp
    5. Click Next



    1. Type your project's GroupId
    2. Type your project's ArtifactId
    3. Click Next



  3. Keep everything as default. Click Next


    1. Type your project's name
    2. Define your project location
    3. Click Finish



  5. Wait for IntelliJ to install Maven and JUnit dependencies


  6. Open web.xml and make changes as shown below in the figure.


  7. It will look like this when you are done.


    1. Click on File > Project Structure
    2. Click on Modules
    3. Click on Dependencies
    4. Click on the + sign as shown in the figure below



    1. Click on Library
    2. Click on triangle arrow next to Application Server Libraries to expand the list
    3. Choose your Tomcat version, I'm choosing 9.0.19



  10. This is optional, but if you are creating Maven, AngularJS, Java web application then you can add Gson jar file to convert your Java objects to JSON.


  11. Click on the check-boxes of Tomcat (and in my case, Gson jar file too), click Apply


    1. Click on Artifacts
    2. Choose your projectname:war exploded
    3. Click OK



    1. Expand src folder
    2. Expand main folder
    3. Right click on main folder, choose New, choose New Directory
    4. Type java



  14. Right click on the new directory named java you just created in the previous step. Choose Mark Directory As > Sources Root. You will notice the folder change its grey color to blue color
    After you mark this folder as sources root, then you can create your domain folder, for example, mine is com.herokuapp.qctune, then you can create sub-directories for your MVC folders, and all of your Java classes, interfaces, servlets will go into java (source root). Your JSP files, CSS, JS, and other files will go to webapp folder, under main as well.


  15. Go to web browser and search for Maven Servlet


  16. Click on the staple Maven servlet version. Here I'm choosing 2.5


  17. Copy the dependency as shown in the figure below and paste it in your pom.xml file


  18. Local dependencies in your pom.xml, and your denpendency will go inside dependencies tag


  19. This is what it looks like after you are done adding your servlet Maven dependency


    1. Click on Add Configuration
    2. Click on + sign and locate Tomcat Server
    3. Click on Tomcat Server and choose Local



    1. Type in your Tomcat server name. I named mine Tomcat Local
    2. Open browser > After launch. By default, IntelliJ will show your default browser when you configure your IntelliJ IDEA, I choose Chrome, but you can choose whichever browser you want Tomcat Localhost to deploy to.



    1. Click on Deployment tab
    2. Click on FIX
    3. Choose your projectname:war exploded



  23. Click OK


    1. Click on File > Project Structure
    2. You can add your Database connector from here using Maven repository or you can add dependency from Maven website to your pom.xml.
    3. Notice I'm using 8.0.16 MySQL Connector and MySQL Workbench for my database



    1. Click on Modules
    2. Click on Web under your project's name as shown in the figure below
    3. Click on the check-box Source Roots
    4. Click OK



  26. If you run into any issue, try to re-configure/reimport Maven from your pom.xml file
    1. Right click on pom.xml
    2. CLick on Maven
    3. Click on Reimport
    4. Click on Tomcat local > restart server


  27. Tomcat will locate index.jsp and will run it
    on your browser, you will see localhost:8080/yourprojectname_war_exploded/index.jsp
    To open servlet, type your servlet name as follow:
    localhost:8080/yourprojectname_war_exploded/YourServlet

Maven doesn't work well with Angular/AngularJS, if you are working with Angular(2-8, upcoming version 9.0.0 is set to release around Oct/Nov 2019), I suggest Spring MVC/Spring Boot
